# 整体流程规格

本文档定义 `dtc` 的端到端处理流程，后续实现应以此为准。

## 目标

`dtc` 从一个配置文件启动，执行入口数据脚本，构建最终全局对象，找到符合规则的模板文件，选出与模板文件名匹配的数据对象，用 EJS 渲染后写入目标输出文件。

## 处理流程

1. 读取配置文件。
2. 解析 `data` 字段，定位入口 JS 文件。
3. 执行入口 JS，并在执行期间支持 `include()`、`remove()`、`update()` 与 `get()` 访问或修改全局对象。
4. 如果配置了 `debugDataOut`，在补充对象元信息之前输出最终全局对象 JSON。
5. 获得最终全局对象后，为每个普通对象补充 `name` 字段。
6. 按配置中的 `tpl` 列表逐项处理模板任务。
7. 对每个模板任务，展开 `files` 中的通配符表达式，得到模板文件列表。
8. 读取每个模板文件内容。
9. 在最终全局对象中递归搜索所有同时满足 `enable === true` 且包含 `match` 的对象。
10. 选出 `match` 与当前模板文件名匹配的对象集合。
11. 如果配置了 `debugMatchOut`，输出模板文件路径和命中的对象数据。
12. 对每个匹配对象执行一次 EJS 渲染。
13. 将同一个模板任务产生的所有渲染结果按固定顺序拼接，写入 `out` 指定的输出文件。

## 核心约束

- 数据对象在模板处理开始前必须已经完全构建完成。
- `debugDataOut` 必须在注入 `name` 之前输出。
- 关系字段补充必须在模板搜索前完成。
- 模板渲染阶段不再执行额外的 `include()` 或 `remove()`。
- 同一个 `tpl` 任务只产生一个输出文件。
- 一个模板文件可以命中多个数据对象，因此会被渲染多次。
- 多个模板文件也可以共同写入同一个输出文件。

## 路径基准

- 配置文件中的相对路径都相对于配置文件所在目录解析。
- `data` 相对于配置文件所在目录。
- `tpl[].files` 中的每个模式相对于配置文件所在目录。
- `tpl[].out` 相对于配置文件所在目录。
- `include()` 中的相对路径相对于当前执行中的 JS 文件所在目录。
- Linux 绝对路径以 `/` 开头。
- Windows 绝对路径支持盘符路径和 UNC 路径。
- 内部比较路径时必须先经过统一规范化，避免因路径分隔符不同导致重复或误判。

## 顺序规则

为保证输出稳定，所有处理顺序必须固定：

1. `tpl` 按配置中声明顺序执行。
2. 单个 `tpl` 任务中的 `files` 模式按声明顺序展开。
3. 每个模式展开后的文件列表按字典序升序排序。
4. 数据对象搜索使用深度优先、从左到右的遍历顺序，只遍历普通对象，不进入数组内部。
5. 单个模板文件命中的数据对象，按搜索得到的顺序渲染。

## 对象元信息与渲染上下文

最终全局对象构建完成后，需要对所有普通对象补充以下字段：

- `name`
  - 根对象的默认值为 `root`。
  - 其他对象的默认值为其在父对象中的键名。
  - 如果对象已经存在 `name` 字段，则保持原值不变。

另外，模板渲染阶段需要提供一个临时的 `parent` 变量：

- `parent` 指向当前命中对象的父对象。
- 根对象作为渲染对象时，`parent` 为 `undefined`。
- `parent` 只存在于渲染上下文中，不写回最终全局对象本身。

## 输出规则

- 单个模板渲染产生一个字符串片段。
- 同一个 `out` 文件内的多个片段按处理顺序拼接。
- 片段之间插入一个换行符 `\n`。
- 如果某个模板文件没有匹配到任何数据对象，则跳过该模板文件，不报错。
- 如果某个 `tpl` 任务最终没有产生任何片段，则生成空文件。

## 调试输出规则

- 如果配置了 `debugDataOut`，输出值为最终全局对象在注入 `name` 之前的 JSON 快照。
- 如果配置了 `debugMatchOut`，输出值为 JSON 数组。
- `debugMatchOut` 中每个元素至少包含：
  - `templatePath`
  - `matchedObjects`
- `matchedObjects` 中如果有多个对象，按模板命中顺序依次输出。

## 错误处理

以下情况必须直接失败并返回非零退出状态：

- 配置文件不存在或格式错误。
- `data` 指向的入口文件不存在。
- 模板模式展开失败。
- 模板文件读取失败。
- 入口 JS 执行失败。
- EJS 渲染抛出异常。
- 输出文件写入失败。

## 非目标

当前版本不包含以下能力：

- 不支持 Node.js 风格插件系统。
- 不支持浏览器环境专用 API。
- 不支持模板热重载。
- 不支持增量构建缓存。
