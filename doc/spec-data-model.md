# 数据模型与脚本执行规格

本文档定义入口数据脚本、`include()`、`remove()`、`replace()`、`update()`、`get()` 和最终全局对象的构建规则。

## 基本模型

- 每个数据文件都是一个 ES Module。
- 每个数据文件可以选择导出一个默认对象，也可以只执行副作用逻辑。
- `dtc` 从配置中的 `data` 入口文件开始执行。
- 所有数据文件共同构建一个最终全局对象，供后续模板阶段使用。

## 文件执行顺序

1. 加载入口文件。
2. 执行入口文件顶层代码。
3. 若顶层代码调用 `include()`，立即处理被包含文件。
4. 顶层代码中的 `remove()`、`replace()`、`update()` 与 `get()` 立即作用于当前全局对象。
5. 当前文件顶层执行结束后，如果存在默认导出，则将其合并到全局对象。
6. 入口文件及其递归包含的所有文件处理完成后，得到最终全局对象。

## `include(path)`

### 作用

加载另一个数据文件，并在其执行完成后把默认导出对象合并进当前全局对象。

### 路径规则

- 相对路径相对于当前执行文件所在目录解析。
- 绝对路径直接使用。
- Linux 与 Windows 的绝对路径判断规则与 `doc/spec-config.md` 保持一致。

### 行为规则

- `include()` 在顶层执行阶段立即生效。
- 同一个绝对路径文件只执行一次。
- 如果出现循环包含，第二次遇到已在加载栈中的文件时直接报错。
- 被包含文件执行完成后，如果存在默认导出，则按合并规则并入全局对象。

## `remove(path)`

### 作用

从当前全局对象中删除指定路径上的字段。

### 路径格式

- `remove()` 接受点分路径字符串，例如 `test.test1`。
- 不支持数组索引。
- 不支持通配符。

### 行为规则

- 如果路径存在，则删除最终字段。
- 如果路径不存在，则忽略，不报错。
- 删除后父对象保持存在，不自动清理空对象。

## `replace(path, value)`

### 作用

直接替换当前全局对象上指定路径的值。

### 路径格式

- `replace()` 接受点分路径字符串，例如 `meta.version`。
- 不支持数组索引。
- 不支持通配符。

### 行为规则

- 如果最终字段已经存在，则直接替换为新值。
- 如果中间路径不存在，则自动创建普通对象。
- 如果中间路径存在但不是普通对象，则直接报错。
- `value` 可以是任意可放入全局对象的 JavaScript 值。

## `update(path, patchObject)`

### 作用

把一个普通对象补丁深合并到当前全局对象上的指定路径。

### 路径格式

- `update()` 接受点分路径字符串，例如 `meta` 或 `modules.detail`。
- 不支持数组索引。
- 不支持通配符。

### 行为规则

- `patchObject` 必须是普通对象。
- 如果目标路径不存在，则自动创建普通对象后再合并。
- 如果中间路径存在但不是普通对象，则直接报错。
- 如果目标路径存在但不是普通对象，则直接报错。
- 对象补丁的合并规则与默认导出对象的深合并规则一致。

## `get(path)`

### 作用

读取当前全局对象上的指定路径，并返回该路径对应的值。

### 路径格式

- `get()` 接受点分路径字符串，例如 `meta.version`。
- 支持读取数组项，例如 `items.0.name`。
- 不支持默认值参数。
- 不支持通配符。

### 行为规则

- 如果路径存在，则返回该路径上的值。
- 如果路径不存在，则直接报错。
- 如果路径中包含数组，则对应路径段必须是数字下标。
- 如果遍历过程中遇到非对象、非数组值但路径还未结束，则直接报错。
- 如果返回值是对象或数组，调用方可以直接修改它，变更会立即反映到当前全局对象。

## 合并规则

合并统一使用深合并，规则如下：

- 目标位置原本不存在时，直接写入新值。
- 两边都是普通对象时，按键递归合并。
- 两边都是数组时，整体覆盖旧值，不做拼接。
- 两边都是标量且 `typeof` 相同，使用新值覆盖旧值。
- 两边都是 `null` 时，结果保持为 `null`。
- 如果两边类型不匹配，立即报错，错误信息必须包含对象路径，并说明对象不匹配，无法合并。
- 不允许把函数、类实例或不可序列化对象并入全局对象。

## 对象元信息补充

最终全局对象构建完成后，必须递归为所有普通对象补充对象元信息：

- `name`
  - 根对象的默认值为 `root`。
  - 非根对象的默认值为它在父对象中的键名。
  - 如果对象已经存在 `name` 字段，则保持原值不变。

`parent` 不应写回最终全局对象本身，而是在模板渲染阶段按遍历关系临时提供。

## 全局对象调试输出

如果配置了 `debugDataOut`，应在补充 `name` 之前输出最终全局对象的 JSON 快照。

原因：

- 不把 `parent` 写回对象可以避免循环引用。
- 在补充 `name` 前输出可以得到更接近原始构建结果的快照。

## 默认导出要求

- 数据文件可以没有默认导出。
- 如果存在默认导出，则默认导出必须是普通对象。
- 如果默认导出存在但不是普通对象，则报错并停止执行。

## 示例

```js
// sub.js
export default {
  test: {
    test1: 2,
    test2: {
      test3: "kkk"
    }
  }
};
```

```js
// root.js
include("sub.js");
remove("test.test1");
const titleBeforeUpdate = get("test.test2.test3");
const secondName = get("items.1.name");
replace("test.test2.test3", "from-update");
update("meta", {
  version: "1.0.0",
  previousTitle: titleBeforeUpdate,
  secondName
});

export default {
  items: [
    { name: "first" },
    { name: "second" }
  ],
  meta: {
    extra: "created"
  },
  test: {
    test2: {
      test3: "45555"
    }
  }
};
```

最终全局对象为：

```js
{
  meta: {
    version: "1.0.0",
    extra: "created",
    previousTitle: "kkk",
    secondName: "second"
  },
  items: [
    { name: "first" },
    { name: "second" }
  ],
  test: {
    test2: {
      test3: "45555"
    }
  }
}
```

## 数据搜索范围

模板阶段会在最终全局对象中递归搜索候选数据对象，规则如下：

- 仅遍历普通对象。
- 不进入数组内部，数组中的对象元素不参与遍历。
- 只有同时满足以下条件的对象才会进入模板匹配阶段：
  - 包含 `match` 字段，且 `match` 为字符串
  - 包含 `enable` 字段，且 `enable === true`
- 候选对象在渲染时保持原始结构，不额外裁剪字段。

## 当前实现模块

- `src/app/data/data-loader.js`：负责入口执行和递归 `include()`。
- `src/app/data/replace-path.js`：负责点分路径直接替换。
- `src/app/data/merge.js`：负责对象深合并。
- `src/app/data/object-meta.js`：负责补充 `name`。
- `src/app/data/remove-path.js`：负责点分路径删除。
- `src/app/data/update-path.js`：负责点分路径对象更新与新增。
- `src/app/data/get-path.js`：负责点分路径读取。
- `src/app/data/data-query.js`：负责递归搜集带 `match` 的对象。
