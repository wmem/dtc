# 数据模型与脚本执行规格

本文档定义入口数据脚本、`include()`、`remove()`、`update()` 和最终全局对象的构建规则。

## 基本模型

- 每个数据文件都是一个 ES Module。
- 每个数据文件都应默认导出一个对象。
- `dtc` 从配置中的 `data` 入口文件开始执行。
- 所有数据文件共同构建一个最终全局对象，供后续模板阶段使用。

## 文件执行顺序

1. 加载入口文件。
2. 执行入口文件顶层代码。
3. 若顶层代码调用 `include()`，立即处理被包含文件。
4. 顶层代码中的 `remove()` 与 `update()` 立即作用于当前全局对象。
5. 当前文件顶层执行结束后，将其默认导出的对象合并到全局对象。
6. 入口文件及其递归包含的所有文件处理完成后，得到最终全局对象。

## `include(path)`

### 作用

加载另一个数据文件，并把其默认导出的对象合并进当前全局对象。

### 路径规则

- 相对路径相对于当前执行文件所在目录解析。
- 绝对路径直接使用。
- Linux 与 Windows 的绝对路径判断规则与 `doc/spec-config.md` 保持一致。

### 行为规则

- `include()` 在顶层执行阶段立即生效。
- 同一个绝对路径文件只执行一次。
- 如果出现循环包含，第二次遇到已在加载栈中的文件时直接报错。
- 被包含文件执行完成后，其默认导出对象按合并规则并入全局对象。

## `remove(path)`

### 作用

从当前全局对象中删除指定路径上的字段。

### 路径格式

- `remove()` 接受点分路径字符串，例如 `test.test1`。
- 不支持数组索引。
- 不支持通配符。

### 行为规则

- 如果路径存在，则删除最终字段。
- 如果路径不存在，则忽略，不报错。
- 删除后父对象保持存在，不自动清理空对象。

## `update(path, value)`

### 作用

更新当前全局对象上的指定路径；如果目标路径不存在，则创建并写入新值。

### 路径格式

- `update()` 接受点分路径字符串，例如 `meta.version`。
- 不支持数组索引。
- 不支持通配符。

### 行为规则

- 如果最终字段已经存在，则直接覆盖其值。
- 如果中间路径不存在，则自动创建普通对象。
- 如果中间路径存在但不是普通对象，则直接报错。
- `value` 可以是任意可放入全局对象的 JavaScript 值。

## 合并规则

合并统一使用深合并，规则如下：

- 目标位置原本不存在时，直接写入新值。
- 两边都是普通对象时，按键递归合并。
- 两边都是数组时，整体覆盖旧值，不做拼接。
- 两边都是标量且 `typeof` 相同，使用新值覆盖旧值。
- 两边都是 `null` 时，结果保持为 `null`。
- 如果两边类型不匹配，立即报错，错误信息必须包含对象路径，并说明对象不匹配，无法合并。
- 不允许把函数、类实例或不可序列化对象并入全局对象。

## 关系字段补充

最终全局对象构建完成后，必须递归为所有普通对象补充关系字段：

- `name`
  - 根对象的默认值为 `root`。
  - 非根对象的默认值为它在父对象中的键名。
  - 如果对象已经存在 `name` 字段，则保持原值不变。
- `parent`
  - 指向当前对象的父对象。
  - 根对象的 `parent` 为 `undefined`。

这些字段在最终全局对象上可见，并供模板阶段直接访问。

## 全局对象调试输出

如果配置了 `debugDataOut`，应在补充 `name` / `parent` 之前输出最终全局对象的 JSON 快照。

原因：

- 补充 `parent` 后对象树会形成循环引用。
- 在补充前输出可以直接得到可序列化 JSON。

## 默认导出要求

- 每个数据文件都必须存在默认导出。
- 默认导出必须是普通对象。
- 如果默认导出不是对象，则报错并停止执行。

## 示例

```js
// sub.js
export default {
  test: {
    test1: 2,
    test2: {
      test3: "kkk"
    }
  }
};
```

```js
// root.js
include("sub.js");
remove("test.test1");
update("test.test2.test3", "from-update");
update("meta.version", "1.0.0");

export default {
  meta: {
    extra: "created"
  },
  test: {
    test2: {
      test3: "45555"
    }
  }
};
```

最终全局对象为：

```js
{
  meta: {
    version: "1.0.0",
    extra: "created"
  },
  test: {
    test2: {
      test3: "45555"
    }
  }
}
```

## 数据搜索范围

模板阶段会在最终全局对象中递归搜索候选数据对象，规则如下：

- 仅遍历普通对象。
- 不进入数组内部，数组中的对象元素不参与遍历。
- 只有同时满足以下条件的对象才会进入模板匹配阶段：
  - 包含 `match` 字段，且 `match` 为字符串
  - 包含 `enable` 字段，且 `enable === true`
- 候选对象在渲染时保持原始结构，不额外裁剪字段。

## 建议的内部模块

- `src/core/data-loader.js`：负责入口执行和递归 `include()`。
- `src/core/merge.js`：负责对象深合并。
- `src/core/object-meta.js`：负责补充 `name` 与 `parent`。
- `src/core/remove-path.js`：负责点分路径删除。
- `src/core/update-path.js`：负责点分路径更新与新增。
- `src/core/data-query.js`：负责递归搜集带 `match` 的对象。
